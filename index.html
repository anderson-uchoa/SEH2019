On Gamifying an Existing Healthcare System: Method, Conceptual Model and Evaluation


This website contains complementary data to the paper.

Abstract

Software gamification aims to make users gradually engaged with system features. Gamifying a system means incorporating game elements, e.g., rankings, and rules into that system. Gamification can be either designed from the earliest development phases, which the literature already supports, or incorporated into an existing system. Although the latter is a market trend, developers lack methods for guiding the gamification of existing systems. Along with software gamification, developers have to define a conceptual model for representing the relations of game elements and rules. Systems that share a common domain may reuse this model. Unfortunately, there is no comprehensive conceptual model for gamifying healthcare systems. In this paper, we introduce a method adapted from a previous work and successfully used to gamify the VazaZika system. This system aims to help to prevent mosquito-borne diseases in economically emerging countries. We also present a conceptual model built on 12 game elements and 12 rules inspired by successful gamified systems. We evaluated our model with 20 citizens by empirically assessing the VazaZika ease to use and user engagement, in terms of fun and motivation. Our results suggest the success of our model in easily engaging users. Both method and model may be reused by other healthcare systems.
